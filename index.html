Comandos git basicos

Aqui estão alguns comandos básicos do Git que são essenciais para o controle de versão:

Configuração Inicial

Configura seu nome de usuário: git config --global user.name "Seu Nome"

Configura seu e-mail: git config --global user.email "seu.email@example.com"

Criação de Repositório

Inicializa um novo repositório Git: git init

Clona um repositório existente: git clone

Estado do Repositório

Verifica o status dos arquivos no diretório: git status

Mostra as diferenças entre o diretório de trabalho e a área de stage: git diff

Mostra o histórico de commits: git log

Manipulação de Arquivos

Adiciona arquivos à área de stage: git add "arquivo"

Adiciona todos os arquivos à área de stage: git add .

Remove arquivos do repositório e do sistema de arquivos: git rm "arquivo"

Remove arquivos da área de stage, mas mantém no sistema de arquivos: git rm --cached "arquivo"

Commits

Faz commit dos arquivos na área de stage: git commit -m "Mensagem do commit"

Faz commit pulando a área de stage: git commit -a -m "Mensagem do commit"

Branches

Lista todas as branches: git branch

Cria uma nova branch: git branch "nome-da-branch"

Muda para outra branch: git checkout "nome-da-branch"

Cria e muda para uma nova branch: git checkout -b "nome-da-branch"

Mescla uma branch com a branch atual: git merge "nome-da-branch"

Revertendo Alterações

Reverte modificações no diretório de trabalho git checkout -- "arquivo"

Reseta a área de stage para o último commit git reset "arquivo"

Reseta a área de stage e o diretório de trabalho para o último commit git reset --hard

Cria um novo commit que reverte um commit específico git revert "commit"

Logs e Histórico

Mostra o log de commits de forma compacta git log --oneline

Mostra o log de commits com gráfico git log --graph

Mostra o log de commits de uma branch específica git log "nome-da-branch "nome-da-branch"

Tags

Cria uma nova tag git tag "nome-da-tag"

Lista todas as tags git tag

Envia tags para o repositório remoto git push origin --tags

Esses comandos cobrem as operações mais comuns que você realizará usando Git. Com o tempo, você pode aprender comandos mais avançados e opções adicionais para personalizar o uso do Git de acordo com suas necessidades.
